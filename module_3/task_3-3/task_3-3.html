Discussion Questions and Answers

  1. What is the difference between invoking and creating a function?

    2. Creating a function means defining it with a specific name and set of instructions, while invoking (or calling) a function means executing that function's instructions by using its name followed by parentheses.

  2. Explain the syntax regarding semicolons and code blocks.

    3. Semicolons are used to terminate statements in JavaScript, although they are sometimes optional. Code blocks are defined by curly braces `{ }` and group multiple statements or lines of code.

  3. What happens when we try to store values from a function when it doesn't return any?

    4. If a function doesn't return a value and we attempt to store its result, the variable will hold `undefined`.

  4. How do we separate parameters from each other when creating a function?

    5. Parameters are separated by commas within the parentheses of the function definition.

  5. Explain how parameter names are related during the invocation and creation of a function.

    6. When creating a function, parameter names are placeholders. When the function is invoked, arguments provided in the call replace these placeholders within the function.

  6. Explain why a parameter is undefined when a function call is missing a parameter.

    7. When a function call is missing a parameter, JavaScript assigns `undefined` to the corresponding parameter within the function, as no value was provided for it.

  7. Explain what happens when we use recursive functions.

    8. Recursive functions call themselves repeatedly until they meet a base condition that stops further calls. Without a base condition, they would cause an infinite loop and eventually a stack overflow error.
