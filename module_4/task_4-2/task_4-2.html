1. What is an array?

    2. An array is a collection of elements that can be of the same or different types, stored in a single variable and indexed for easy access.

  2. What does it mean to "hardcode"?

    3. Hardcoding refers to embedding fixed data or values directly into the source code, instead of fetching them dynamically or through input.

  3. How do we traverse an array without knowing its length?

    4. By using loops, such as a for or forEach loop, which iterates through the array dynamically based on its size.

  4. Why store instances of a class in an array?

    5. Storing instances of a class in an array helps organize and manage multiple objects, enabling easy manipulation and traversal.

  5. How do "Workdays" and "Weekends" differ in EWeekDays?

    6. Workdays include weekdays such as Monday through Friday, while Weekends represent Saturday and Sunday. Each has distinct attributes for functionality.

  6. How does a "callback" function work in sorting?

    7. A callback function is passed to the sort method to define custom sorting logic. It compares two elements and returns a value to determine their order.

  7. What happens in the sorting routines?

    8. The callback function compares array elements pairwise and rearranges them based on the comparison, either in ascending or descending order.

  8. What is the first element of a two-dimensional array?

    9. In a 2D array, the first element is typically the first row, which is itself an array containing all the elements of that row.
